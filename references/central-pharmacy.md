# 中心药房专项参考库

> 本文档为"中心药房解决方案"的深度参考库，配合 `references/scenarios.md` 场景六使用。涵盖业务模型、数据模型、接口规范、关键集成、厂商短名单、合规要点与常见踩坑。当方案涉及医共体 / 医联体 / 医疗集团统一药房、处方流转、药品追溯、慢病长处方、自动发药时加载本文件。

## 一、业务模型详解

### 1.1 核心业务域

| 业务域 | 子域 | 关键能力 |
|--------|------|---------|
| **药品主数据** | 字典管理 | 对接国家医保统一编码、自维护药品字典、供应商目录 |
| **统一采购** | 招标议价 | 集中招标、合同管理、议价谈判 |
| | 采购执行 | 采购计划、订单、入库验收、退货 |
| | 对账结算 | 三方对账（医院-供应商-集团）、发票匹配 |
| **库存管理** | 多级库存 | 中心药房总库 → 成员单位分库 → 科室柜 |
| | 调拨配送 | 成员单位间调拨、患者终端配送 |
| | 效期管理 | 近效期预警、过期锁定、先入先出 |
| **处方流转** | 处方接收 | 来自 HIS / EMR / 互联网医院 |
| | 审方前置 | 对接云审方中心（见 `references/cloud-prescription.md`） |
| | 调剂发药 | 院内自动发药、合作药店调剂、第三方物流 |
| | 用药跟踪 | 签收、用药提醒、不良反应上报 |
| **药品追溯** | 一物一码 | 入库扫描、出库扫描、终端核验 |
| | 平台对接 | 国家药品追溯码平台（码上放心等） |
| **慢病长处方** | 续方管理 | 慢病目录、长处方模板、有效期监控 |
| | 配送协同 | 第三方物流对接、冷链管理（胰岛素等） |
| **自动发药** | 门诊发药 | 自动发药机对接 HIS |
| | 住院摆药 | 单剂量摆药机对接 EMR |

### 1.2 处方流转全链路

```
医生开方（HIS/EMR/互联网医院）
    ↓ 处方上传（FHIR MedicationRequest）
中心药房处方接收
    ↓ 调用云审方中心
自动审方 + 药师复审
    ↓ 审方通过
调剂路由决策
    ├─ 院内药房 → 自动发药机 → 患者自取
    ├─ 合作药店 → 药店调剂 → 患者自取 / 配送
    └─ 第三方物流 → 配送上门 → 签收
    ↓
用药跟踪与反馈
    ├─ 签收确认
    ├─ 用药提醒（短信/微信）
    └─ 不良反应上报（ADR）
```

## 二、数据模型与接口规范

### 2.1 药品主数据核心字段

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| 药品本位码 | String(14) | 国家药品本位码 | 国家药监局 |
| 国家医保编码 | String(9) | 国家医保统一编码 | 国家医保局 |
| 通用名 | String | 通用名 | 国家药监局 |
| 商品名 | String | 商品名 | 厂商 |
| 剂型 | Enum | 片剂/胶囊/注射剂/... | 标准字典 |
| 规格 | String | 如 0.25g×24片 | 厂商 |
| 包装规格 | String | 大/中/小包装换算 | 厂商 |
| 生产厂家 | String | 厂商全称 | 厂商 |
| 批准文号 | String | 国药准字 HXXXXXXXX | 国家药监局 |
| 处方药分类 | Enum | Rx / OTC / 麻精 / 抗菌分级 | 标准 |
| ATC 编码 | String | 解剖-治疗-化学分类 | WHO ATC |
| 是否基药 | Bool | 国家基本药物目录 | 国家卫健委 |
| 是否集采 | Bool | 集中带量采购品种 | 国家医保局 |
| 供应商 | String | 供应商主数据关联 | MDM |
| 中标价 | Decimal | 集采中标价 / 议价 | 采购系统 |
| 零售价 | Decimal | 零售价 | 采购系统 |
| 库存上下限 | Integer | 安全库存阈值 | 库存管理 |

### 2.3 关键接口清单

| 接口 | 协议 | 方向 | 说明 |
|------|------|------|------|
| 处方上传 | FHIR RESTful POST /MedicationRequest | HIS → 中心药房 | 处方流转入口 |
| 审方结果回传 | FHIR RESTful PUT /MedicationRequest | 云审方 → 中心药房 | 审方结果 |
| 调剂状态回传 | RESTful /dispense-status | 中心药房 → HIS | 调剂进度 |

## 三、关键集成点

### 3.1 与 HIS / EMR 集成

- **处方上传**：HIS 开方后通过集成平台转发至中心药房
- **库存查询**：HIS 实时查询药品库存（避免开方后无药）
- **调剂状态**：调剂完成后回传 HIS，触发患者通知
- **结算联动**：调剂完成触发医保结算

### 3.2 与互联网医院集成

- **复诊开方**：互联网医院复诊处方流转至中心药房
- **配送选择**：患者选择药店自取 / 配送上门
- **处方有效期**：互联网医院处方有效期 7 天

### 3.3 与国家药品追溯平台对接

| 平台 | 对接方式 | 数据内容 |
|------|---------|---------|
| 码上放心（阿里健康） | API + 二维码扫描 | 药品追溯码、出入库记录 |
| 国家医保药品追溯 | API | 医保结算关联追溯码 |
| 厂商自有追溯 | API | 厂商批次追溯 |

### 3.4 与医保系统对接

- **医保电子凭证**：扫码识别患者身份
- **医保结算接口**：国家医保局统一接口（Vue 版）
- **DRG/DIP 关联**：药品费用计入病组结算

### 3.5 与物流系统对接

- **院内物流**：自建配送（护士站配送）
- **第三方物流**：顺丰、京东物流医药专线
- **冷链管理**：胰岛素、生物制品温控监测

## 五、合规要点

### 5.1 资质合规

- 中心药房须依托具有《医疗机构执业许可证》的实体机构
- 处方流转须依托互联网医院或实体医院资质
- 零售药店参与调剂须具备《药品经营许可证》

### 5.2 处方管理

- 普通处方有效期 **7 天**
- 急诊处方有效期 **1 天**
- 慢病长处方有效期最长 **12 周**
- 麻精药品**不参与流转**，必须院内调配
- 处方保存 **≥15 年**（电子病历保存要求）

### 5.3 药品流通合规

- 药品须一物一码追溯，对接国家追溯平台
- 冷链药品全程温控记录
- 临近效期药品（<6 个月）须标识并优先使用
- 过期药品须按医疗废物处理，记录销毁

### 5.4 医保合规

- 医保目录内药品方可医保结算
- 集采品种须优先使用
- 处方与诊断须匹配（医保规则审核）
- 严禁虚开处方、套保骗保

## 六、常见踩坑

### 6.1 药品主数据混乱

- **问题**：各单位药品字典不一致，导致处方流转失败
- **应对**：建设集团统一 MDM，强制对接国家医保统一编码

### 6.2 处方流转跨机构结算失败

- **问题**：A 院开方、B 药店调剂，医保结算归属争议
- **应对**：明确"开方机构结算"或"调剂机构结算"规则，提前与医保局沟通

### 6.3 自动发药机卡药

- **问题**：异形药品、软包装卡药，发药失败率高
- **应对**：建立"可上机药品目录"，非目录药品走人工窗口

### 6.4 药品追溯码扫描失败

- **问题**：包装码印刷质量差、破损，扫描失败率高
- **应对**：多角度扫描 + 人工补录，建立异常处理流程

### 6.5 慢病长处方管理漏洞

- **问题**：长处方续方未管控有效期，导致超期开药
- **应对**：长处方有效期硬管控，到期自动停开

### 6.6 第三方物流冷链断链

- **问题**：胰岛素等冷链药品配送途中温控中断
- **应对**：温控记录仪全程监测，断链药品拒收

### 6.7 集采品种库存积压

- **问题**：集采品种强制采购量，但临床使用率不达预期
- **应对**：集采品种使用率监控，与科室绩效挂钩

## 七、典型部署架构

### 7.1 集团化部署（推荐）

```
┌──────────────────────────────────────────────────────────────┐
│ 集团中心药房云平台（多租户 SaaS）                              │
│  - 药品主数据 / 采购 / 库存 / 处方流转 / 追溯                  │
├──────────────────────────────────────────────────────────────┤
│ 集成层：FHIR / 医保接口 / 国家追溯平台 / 物流                  │
├──────────────────────────────────────────────────────────────┤
│ 成员单位 1   成员单位 2   ...   成员单位 N                    │
│ HIS/EMR      HIS/EMR      ...   HIS/EMR                       │
│ 自动发药机   药店终端      ...   自动发药机                     │
└──────────────────────────────────────────────────────────────┘
```

### 7.2 单院部署

中心药房系统本地化部署，仅对接本院 HIS/EMR 与国家追溯平台，适用于单体大型医院。

### 7.3 区域化部署

由区域卫健委牵头，为区域内所有医疗机构提供中心药房服务，适用于县域医共体 / 城市医疗集团。
