# DRG/DIP 2.0 专项知识库

> 本文件为「医疗信息化解决方案专家 Skill」的 DRG/DIP 专项补充库。当用户方案涉及医保支付方式改革、医保智能审核、病案首页质控、成本核算、绩效考核或医院精细化运营时加载。
>
> **版本提示**：本文以 **DRG/DIP 付费 2.0 版分组方案**（国家医保局 2024-07-23 发布）为当前实施基准。3.0 版调整工作已于 2026 年启动，尚未正式发布；编撰方案时若涉及 3.0，须用 WebSearch 核验最新进展，避免在方案中写死未生效的版本号。

## 1. 概念与背景

### 1.1 什么是 DRG / DIP

- **DRG（Diagnosis Related Groups，按病组付费）**：按「主要诊断 + 并发症/合并症 + 手术操作 + 年龄/性别」等要素，将病例分入若干病组，每组给定固定支付标准（打包付费）。
- **DIP（Big Data Diagnosis-Intervention Packet，按病种分值付费）**：基于大数据分析，按「主要诊断 + 主要操作」组合赋分值，年终按区域总分值和点值结算。
- 两者本质都是**按病种付费**，从「按项目付费」转向「价值/打包付费」，目的是规范诊疗行为、控制不合理费用、提升基金使用效率。

### 1.2 改革进程

| 时间节点 | 事件 | 方案编撰引用要点 |
|---------|------|----------------|
| 2019 | DRG 1.0 版（CHS-DRG）分组方案发布 | 历史版本，已淘汰 |
| 2021 | DIP 1.0 版病种库发布 | 历史版本，已淘汰 |
| 2024-07-23 | **《关于印发按病组（DRG）和病种分值（DIP）付费 2.0 版分组方案并深入推进相关工作的通知》**发布 | 现行基准 |
| 2024-12-31 前 | 已开展地区完成 2.0 切换准备 | 切换期要求 |
| 2025 起 | 各统筹地区统一使用 2.0 版分组 | 实施基准年 |
| 2025 年底 | 2.0 版基本在全国所有统筹地区全面落地 | 落地完成 |
| 2026 | 3.0 版调整工作启动（趋势，未发布） | 编撰时核验 |

### 1.3 方案编撰定位

DRG/DIP 不是独立系统，而是一套**贯穿病案、临床、医保、运营的数据治理与决策体系**。在方案中通常作为以下场景的核心子模块出现：

- 智慧医院「智慧管理」「运营决策」板块
- 城市医疗集团 / 县域医共体「医保协同」板块
- 医院 HRP / 成本核算系统升级的驱动因素
- 病案统计、质控系统的升级理由

## 3. DRG/DIP 2.0 版核心变化（硬事实，可直接引用）

### 3.1 DRG 2.0 分组数据

| 指标 | 1.0 版 | 2.0 版 | 变化 |
|------|--------|--------|------|
| 核心分组（ADRG） | 376 组 | **409 组** | +33（外科手术 182、非手术室操作 37、内科诊疗 190） |
| 细分组（DRGs） | 628 组 | **634 组** | 微调 |
| 重点优化学科 | — | 13 个学科（重症医学、血液免疫、肿瘤、烧伤、口腔颌面外科等） | 新增 |
| 新增核心组别 | — | 26 个 | 新增 |

优化方向：回应临床关切，解决复杂重病入组难、补偿不合理；减少 QY（歧义）组和 0000 组（未入组）产生；MCC/CC 表测算引入遗传算法、考虑麻醉分级。

### 3.2 DIP 2.0 病种库数据

| 指标 | 1.0 版 | 2.0 版 |
|------|--------|--------|
| 核心病种总数 | 11553 组 | **9520 组**（覆盖 95%+ 出院病例） |
| 保守治疗组 | — | 3209 组 |
| 手术操作组 | — | 6311 组（主要操作 5211 + 主要操作+相关操作 1100） |
| 疾病诊断覆盖 | — | 20 章 / 218 节 / 1133 类目 / 3332 亚目 |

优化方向：病种数量减少、集中度提升；减少"未特指/其他"病种，诊断更明确；采用"主要诊断+主要操作+相关操作"聚类，体现双侧手术、转科等临床价值。

### 3.3 两大机制（方案必写，体现政策理解深度）

1. **特例单议机制**：不适合按标准支付的复杂危重病例，医疗机构可向医保经办自主申报。这是 DRG/DIP 应对病情复杂多变的**兜底机制**。方案应说明本院特例单议申报流程、比例管理与资料留存。
2. **除外支付机制**：新药耗、新项目等可在除外支付中体现，避免创新技术因控费被压制。方案应说明对创新技术（如新术式、新耗材）的支付衔接。
3. **预付金机制**：医保可向管理规范、信用良好的机构预付约 1 个月预付金，缓解资金压力。
4. **绩效禁限**：通知明确**医疗机构不得将 DRG/DIP 支付标准作为限额对医务人员考核或与绩效分配挂钩**——方案中涉及绩效考核设计时须规避此红线。

## 4. 业务模型与数据流

### 4.1 院端落地总体流程

```
病案首页/结算清单（HIS/EMR 生成）
       │  (ICD-10 诊断编码、ICD-9-CM3 手术操作编码、医保 2.0 编码)
       ▼
  病案首页质控（编码审核、逻辑校验、缺项校验）
       │
       ▼
  分组器（DRG/DIP 本地化分组引擎）
       │  → 入组结果、权重/分值、支付标准
       ▼
  医保结算（对接本地医保平台，上传结算清单）
       │
       ▼
  盈亏分析（实际费用 vs 支付标准 → 盈利/亏损病组）
       │
       ▼
  反馈闭环：临床路径优化 / 编码质量提升 / 成本核算调整
```

### 4.2 五大核心落地模块

| 模块 | 作用 | 上游 | 下游 |
|------|------|------|------|
| **分组器** | 按 2.0 规则本地化入组，模拟结算 | 病案首页、医保目录 | 结算、盈亏分析 |
| **病案首页质控** | 编码准确性校验、逻辑校验、DRG 入组预校验 | HIS/EMR | 分组器 |
| **医保智能审核** | 事前提醒+事中审核+事后分析，违规规则库 | 医嘱、费用、病案 | 医保结算 |
| **成本核算** | 按病组/病种核算实际成本，与支付标准对比 | HRP、物资、财务 | 盈亏分析、绩效 |
| **绩效考核** | 基于病组结构、CMI、时间消耗指数等的科室/医生评价（**不挂钩支付限额**） | 分组、成本、运营 | 管理层决策 |

### 4.3 关键数据实体

- **医疗保障基金结算清单**：国家统一标准，替代传统首页用于医保结算
- **ICD-10 / ICD-9-CM3**：诊断与手术操作编码（须与医保 2.0 编码映射）
- **病组权重 / 分值**：DRG 权重、DIP 分值
- **CMI（病例组合指数）**：反映收治病例难度，方案常作为医院能力指标
- **时间消耗指数 / 费用消耗指数**：DRG 评价医疗效率的核心指标

## 5. 与医院信息化系统的集成

### 5.1 对接系统清单

| 对接系统 | 集成内容 | 接口要点 |
|---------|---------|---------|
| HIS | 患者基本信息、费用明细、医嘱 | 获取结算所需全量费用与诊断 |
| EMR | 病案首页、诊断、手术记录 | 首页数据质量决定入组准确性 |
| 病案统计系统 | 首页编码、DRG 预分组 | 编码映射（ICD↔医保 2.0） |
| HRP / 成本核算 | 科室成本、物资消耗 | 按病组归集实际成本 |
| 医保平台 | 结算清单上传、清算、反馈 | 本地医保 API / 中台 |
| 集成平台 | 统一消息编排 | 建议走集成引擎，避免点对点 |

## 7. 合规要点（审核必查）

- [ ] 是否明确以**本地医保局发布的 DRG 细分组 / DIP 病种库**为实施依据（非直接套用国家版数字）
- [ ] 是否规划**病案首页 / 结算清单数据质量**治理（编码准确率、完整率指标）
- [ ] 是否建立**特例单议**申报与管理流程
- [ ] 是否说明**除外支付**对新技术的衔接
- [ ] 绩效考核设计是否**规避"将支付标准作为限额考核医生"**的红线
- [ ] 是否对接**国家统一医疗保障基金结算清单**标准
- [ ] 数据是否涉及患者隐私，是否满足脱敏与权限管控

## 8. 常见踩坑（方案须规避）

1. **直接用国家版分组数当本地结算依据**：各地细分组/病种库有本地化，必须注明以本地版本为准。
2. **病案首页质量被低估**：编码不准确直接导致入组错误、医保拒付，方案须把首页质控当独立模块。
3. **把 DRG/DIP 当"控费工具"写死考核**：政策明确禁止限额考核医生，方案绩效考核板块须正向设计（CMI、效率指数）。
4. **分组器与 HIS 脱节**：分组器若不能实时读取最新费用与诊断，模拟结算失真。
5. **忽略特例单议**：复杂病例若不在方案中规划申报流程，医院实际亏损面扩大。
6. **成本核算缺失**：只做分组不做成本归集，无法回答"哪些病组真盈利"。
7. **2.0/3.0 版本混淆**：3.0 尚未正式发布，方案写"全面采用 3.0"属事实错误。

## 9. 方案编撰融合示例

### 9.1 在「智慧医院方案」中

归入「智慧管理 → 运营决策」板块，章节可命名为「基于 DRG/DIP 2.0 的精细化运营」，内容为：分组器 + 病案质控 + 成本核算 + 绩效（正向）四位一体，目标指标列出 CMI、时间消耗指数、费用消耗指数、病案首页合格率。

### 9.2 在「城市医疗集团 / 县域医共体方案」中

归入「医保协同」板块，强调集团内 DRG/DIP **总额打包、结余留用、特例单议协同**，以及上下级机构病组结构差异带来的资源下沉策略。

### 9.3 在「HRP / 成本核算升级方案」中

以 DRG/DIP 2.0 为驱动理由，说明按病组成本核算对支付改革的支撑，列出 ODR（运营数据中心）与 HRP 的衔接。
