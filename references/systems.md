# 医疗信息化核心系统知识库

> 本文档收录方案编撰中常涉及的核心业务系统职责、数据边界、典型厂商与集成方式。引用时按"系统简称 + 职责边界 + 数据流"组织表述。

## 一、医院核心业务系统全景

医院信息系统生态按照"医、护、技、药、管、辅"分层：

```
┌──────────────────────────────────────────────────────────────┐
│  管理层   HRP | HRPOA | 财务 | 物资 | 智慧后勤 | 决策支持 BI  │
├──────────────────────────────────────────────────────────────┤
│  辅助层   LIS | PACS/RIS | 病理 | 超声 | 内镜 | 麻醉 | 重症   │
├──────────────────────────────────────────────────────────────┤
│  核心层   HIS | EMR | 护理 | 药学 | 手术 | 输血 | 体检        │
├──────────────────────────────────────────────────────────────┤
│  数据层   CDR | ODR | SDR | BI | 数据湖 | 集成平台           │
├──────────────────────────────────────────────────────────────┤
│  患者层   互联网医院 | 互联网诊疗 | 智慧服务 | 全病程管理     │
└──────────────────────────────────────────────────────────────┘
```

## 二、各系统职责与边界

### 1. HIS（Hospital Information System，医院信息系统）

- **职责**：医院运营核心，覆盖挂号、收费、住院结算、医保对接、医嘱管理、患者主索引、字典维护
- **边界**：业务交易为主，临床文书由 EMR 承担；HIS 是"账"，EMR 是"病历"
- **关键数据**：就诊记录、医嘱、费用明细、结算单、患者主索引（EMPI）
- **集成方式**：通过集成平台与 EMR/LIS/PACS/药学交互；医保接口走国家医保统一接口
- **典型厂商**：东华、东软、卫宁、用友医疗、易联众、金蝶医疗、中科软

### 2. EMR（Electronic Medical Record，电子病历系统）

- **职责**：临床诊疗文书撰写与管理，包括入院记录、病程、手术记录、出院小结、知情同意书电子签
- **边界**：聚焦"医生写病历"，与 HIS 共享医嘱；CDR 是其数据沉淀
- **关键数据**：临床文书、生命体征、诊断（ICD-10）、手术（ICD-9-CM-3）、过敏史
- **集成方式**：从 LIS/PACS 接收检查检验结果回填病历；通过共享文档（WS/T 500-2016）对外发布
- **典型厂商**：嘉和、海泰、东软、卫宁、医惠、中联

### 3. LIS（Laboratory Information System，实验室信息系统）

- **职责**：检验申请、标本管理、仪器接口、报告出具、质控
- **边界**：标本流转全过程，与 HIS/EMR 互开申请；不负责收费（HIS 负责）
- **关键数据**：检验申请单、标本条码、仪器原始结果、报告（含参考值与单位）
- **集成方式**：HL7 V2 ORU^R01 上传结果；仪器接口 RS-232 / TCP / ASTM
- **典型厂商**：芯源、瑞美、金蝶、嘉和、东方惠普

### 4. PACS（Picture Archiving and Communication System，医学影像归档与通信系统）

- **职责**：医学影像采集、存储、传输、调阅
- **边界**：影像数据管理，RIS 负责检查预约 / 报告流程
- **关键数据**：DICOM 影像、报告（SR / 结构化报告）
- **集成方式**：DICOM 3.0 采集与传输；IHE SWF 预约流程；XDS-I 跨机构影像共享
- **存储要求**：影像长期在线 / 近线 / 离线分层；按医院规模年新增 30~200 TB
- **典型厂商**：东软、海纳医信、卫宁、医脉通、奕瑞、安科、飞利浦、GE、西门子

### 5. RIS（Radiology Information System，放射信息系统）

- **职责**：检查预约、技师工作站、报告书写、报告审核
- **边界**：放射科业务流程，与 PACS 强耦合
- **集成方式**：与 HIS 通过 HL7 ORM^O01 申请；与 PACS DICOM Worklist 同步

### 6. HRP（Hospital Resource Planning，医院综合运营管理）

- **职责**：人、财、物、设备统一管理，覆盖会计、成本、绩效、物资、固定资产、设备全生命周期
- **边界**：运营管理维度，与 HIS 通过核算接口对账
- **关键数据**：人事档案、物资流水、设备资产、成本分摊、绩效数据
- **集成方式**：与 HIS 财务核算接口、与 OA 审批接口、与设备 IoT 数据采集
- **典型厂商**：用友、金蝶、望海、东软、汉端

### 7. 手术麻醉系统（Anesthesia Information Management System, AIMS）

- **职责**：术前访视、术中监护数据采集、麻醉记录单、术后随访
- **关键数据**：生命体征高频采集、麻醉用药、事件时间轴
- **集成方式**：监护仪 / 麻醉机接口；HL7 上传事件

### 8. 重症监护系统（Critical Care Information System, CCIS）

- **职责**：ICU 患者监护、护理记录、出入量、评分
- **集成方式**：监护仪接口、呼吸机接口、检验结果回填

### 9. 病案管理系统

- **职责**：病案首页、ICD 编码、病案归档、病案借阅
- **关键数据**：ICD-10、ICD-9-CM-3、病案号
- **集成方式**：从 EMR 拉取出院病历；上报区域病案系统

### 10. 体检信息系统

- **职责**：团体/个人体检预约、流程引导、结果汇总、总检报告
- **集成方式**：与 LIS/PACS 互开检查；对接 HIS 收费

## 三、集成平台与数据中心

### 1. 集成平台（Integration Platform / Integration Engine）

- **职责**：统一各业务系统接口，提供消息路由、协议转换、消息队列、统一服务总线（ESB）
- **关键技术组件**：
  - 集成引擎（InterSystems HealthShare / Ensemble、Mirth Connect / NextGen Connect、Rhapsody、Corepoint）
  - ESB / API 网关（Kong、APISIX）
  - 消息中间件（Kafka、RabbitMQ、RocketMQ）
- **核心能力**：协议转换（HL7 V2 ↔ FHIR）、消息路由、协议适配、统一患者主索引（EMPI）、消息审计
- **方案撰写要点**：
  - 必须明确集成引擎选型与替代方案
  - 老旧系统通过适配器接入
  - 互联互通成熟度测评要求平台支持 FHIR RESTful 服务

### 2. CDR（Clinical Data Repository，临床数据中心）

- **职责**：汇聚 EMR、LIS、PACS、护理等临床数据，提供统一临床视图、科研数据服务
- **数据模型**：基于 FHIR 资源或自建临床数据模型；遵循 WS/T 500-2016 共享文档
- **关键能力**：统一临床视图（360 视图）、临床数据检索、科研数据导出、临床决策支持数据底座

### 3. ODR（Operational Data Repository，运营数据中心）

- **职责**：汇聚 HIS、HRP、财务、医保数据，提供运营 BI、绩效分析
- **典型工具**：FineBI、帆软、PowerBI、Tableau、自研数据中台

### 4. SDR（Science Data Repository，科研数据 Repository）

- **职责**：科研数据清洗、标注、伦理审批管理、AI 训练数据集管理
- **关键能力**：单病种数据集、专科数据库、AI 模型训练样本管理

## 四、跨机构 / 区域级系统

| 系统 | 职责 | 标准依据 |
|------|------|---------|
| 区域卫生信息平台 | 居民健康档案、双向转诊、公卫上报 | WS/T 503-2016 |
| 紧密型医共体平台 | 县乡一体化、人财物统一、检查检验互认 | 紧密型县域医共体建设方案 |
| 紧密型城市医疗集团 | 城市内三级医院与社区联动 | 紧密型城市医疗集团建设试点方案 |
| 互联网医院 | 在线复诊、电子处方、药品配送 | 互联网医院管理办法 |
| 区域影像中心 | 跨机构影像调阅、远程诊断 | XDS-I / DICOM |
| 区域检验中心 | 标本流转、结果互认 | LIS + 集成平台 |
| 区域心电中心 | 心电数据汇聚、远程诊断 | DICOM Waveform |
| 区域病理中心 | 病理切片数字化、远程会诊 | DICOM / WSI |

## 五、AI 赋能应用接入位置

| AI 应用 | 接入系统 | 数据流向 | 关键技术 |
|---------|---------|---------|---------|
| 智能导诊 | 互联网医院 / 自助机 | 患者 → AI → 推荐科室 → 挂号 | NLP 意图识别、知识图谱 |
| 医学影像 AI 辅助诊断 | PACS / RIS | DICOM → AI 推理 → SR 结构化报告 | 深度学习（CNN/U-Net/Transformer） |
| 病历结构化 NLP | EMR / 病案系统 | 自由文本 → 实体抽取 → 结构化字段 | NER + 关系抽取 + LLM |
| 临床决策支持 CDSS | HIS / EMR / CDR | 规则引擎 + LLM 推理 → 弹窗提醒 | 规则引擎 + LLM 工具调用 |
| AI 病案质控 | 病案系统 | 病历 → 缺陷检测 → 修改建议 | NLP 分类 + 规则 |
| 智能审方 | 药学系统 / HIS | 处方 → 审方规则 → 风险评级 | 规则引擎 + 知识库 |
| 智能随访 | 互联网医院 | 出院患者 → 自动外呼/短信 → 随访记录 | ASR + NLP |
| 大模型临床科研助手 | SDR | 自然语言查询 → SQL/向量检索 → 结果 | RAG + Text2SQL |
